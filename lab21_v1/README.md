# Лабораторна робота №21

## Тема
OCP: гнучкі алгоритми розрахунку (Factory Method / Strategy)

## Мета роботи
Застосувати принцип відкритості/закритості (Open/Closed Principle) для створення
гнучкої системи розрахунку вартості доставки з використанням патернів
Strategy та Factory Method.

## Завдання
Реалізувати консольний застосунок для розрахунку вартості доставки з можливістю
легкого додавання нових алгоритмів без зміни існуючого коду.

## Використані патерни
- Strategy
- Factory Method

## Опис реалізації

### Інтерфейс IShippingStrategy
Описує спільний метод для всіх алгоритмів доставки:
- CalculateCost(distance, weight)

### Реалізовані стратегії
- StandardShippingStrategy
- ExpressShippingStrategy
- InternationalShippingStrategy
- NightShippingStrategy (додано для демонстрації OCP)

### Factory Method
Клас `ShippingStrategyFactory` створює відповідну стратегію доставки
на основі введеного користувачем типу.

### DeliveryService
Клас не залежить від конкретних реалізацій стратегій, працює лише
з інтерфейсом `IShippingStrategy`.

## Демонстрація принципу OCP
Для додавання нового типу доставки (NightShippingStrategy):
- створено новий клас стратегії
- додано підтримку у фабрику
- клас DeliveryService не змінювався

## Запуск програми
1. Користувач обирає тип доставки
2. Вводить відстань та вагу
3. Програма обчислює та виводить вартість доставки

## Висновок
Принцип відкритості/закритості дотримано, оскільки система легко
розширюється шляхом додавання нових стратегій без зміни існуючої логіки.
