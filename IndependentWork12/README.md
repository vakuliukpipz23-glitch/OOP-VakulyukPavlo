# Самостійна робота №12  
**Тема:** PLINQ: дослідження продуктивності та безпеки  

## Мета роботи

Дослідити переваги та недоліки PLINQ (Parallel LINQ) у порівнянні зі звичайним LINQ,  
виміряти його продуктивність на різних обсягах даних та проаналізувати потенційні проблеми безпеки при паралельній обробці.


## 1. Опис експериментів з продуктивності

### 1.1. Вхідні дані

- Тип колекції: `List<int>`.
- Розміри колекцій:
  - 1 000 000 елементів  
  - 5 000 000 елементів  
  - 10 000 000 елементів  
- Дані: випадкові цілі числа у діапазоні від 1 до 10 000 000.

### 1.2. Обчислювальна операція

Для кожного елемента виконувалося:

1. Перевірка числа на простоту (метод `IsPrime(int n)`).
2. Для простих чисел додатково обчислювався квадратний корінь `Math.Sqrt(n)`.

   var linqResult = data
       .Where(n => IsPrime(n))
       .Select(n => Math.Sqrt(n))
