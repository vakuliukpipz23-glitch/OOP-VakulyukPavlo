# Лабораторна робота №24  
## Використання патернів Strategy та Observer

### Мета роботи

Метою роботи є розробка гнучкої системи обробки числових даних із можливістю динамічної зміни алгоритмів обчислення та автоматичного сповіщення залежних компонентів про результати обробки із використанням патернів Strategy та Observer.

### Опис виконання роботи

У процесі виконання лабораторної роботи було створено консольний застосунок для обробки числових значень із використанням структурних принципів об'єктно-орієнтованого програмування.

Для забезпечення можливості зміни алгоритму обробки даних під час виконання програми було реалізовано патерн Strategy.  
Створено інтерфейс `INumericOperationStrategy`, який визначає метод виконання обчислення для заданого числа.

На основі цього інтерфейсу реалізовано три алгоритми обробки:

- обчислення квадрату числа;
- обчислення кубу числа;
- обчислення квадратного кореня.

Для використання цих алгоритмів створено клас `NumericProcessor`, який містить посилання на поточну стратегію обробки.  
Клас дозволяє змінювати активний алгоритм під час виконання програми за допомогою методу зміни стратегії.  
Таким чином, логіка обчислення ізольована від основної логіки програми, що забезпечує її гнучкість та розширюваність.

Для організації автоматичного сповіщення про результати обчислення було реалізовано патерн Observer.  
Створено клас `ResultPublisher`, який виконує роль видавця та генерує подію після отримання результату обробки.

Було реалізовано кілька спостерігачів:

- спостерігач для виведення результатів у консоль;
- спостерігач для збереження історії обчислень;
- спостерігач для контролю перевищення заданого порогового значення.

Спостерігачі підписуються на подію видавця та автоматично отримують інформацію про результати без прямої взаємодії з обчислювальним модулем.

### Демонстрація роботи

У методі `Main` було створено:

- об'єкт обробника чисел;
- видавця результатів;
- набір спостерігачів.

Спостерігачі підписуються на подію видавця.  
Після цього виконуються обчислення для кількох чисел із використанням різних стратегій.

Після кожного обчислення результат передається видавцю, який ініціює подію та повідомляє всі підписані компоненти.

Таким чином забезпечується:

- можливість динамічної зміни алгоритмів;
- автоматичне реагування системи на результати;
- відсутність жорсткого зв'язування між компонентами.

### Висновок

У ході виконання лабораторної роботи було реалізовано патерни Strategy та Observer для створення гнучкої системи обробки числових даних.

Застосування патерну Strategy дозволило змінювати алгоритми обробки без зміни структури програми.  
Патерн Observer забезпечив автоматичне сповіщення залежних компонентів про результати виконання.

У результаті створено розширювану систему зі слабким зв'язуванням компонентів, що відповідає сучасним принципам проєктування програмного забезпечення.

Поставлена мета роботи досягнута.
