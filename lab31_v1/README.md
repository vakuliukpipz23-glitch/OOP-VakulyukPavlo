# Лабораторна робота №31
## Тестування з Moq

### Мета роботи
Навчитися використовувати бібліотеку Moq для створення mock-об'єктів та тестування класів з залежностями.

# Варіант
OrderService  
Залежності:
- IOrderRepository
- IEmailService

# Опис роботи

Було створено сервіс **OrderService**, який використовує дві залежності:
- репозиторій замовлень
- сервіс відправки email

Залежності передаються через **Dependency Injection (конструктор)**.

# Використання Moq

Для тестування використовувалась бібліотека **Moq**.

Основні методи:

**Setup()**  
використовується для налаштування поведінки mock-об'єкта.

Приклад:

repoMock.Setup(r => r.GetById(1)).Returns(order);

**Verify()**  
перевіряє, чи був викликаний метод.

Приклад:

repoMock.Verify(r => r.Save(order), Times.Once);

# Написані тести

Було реалізовано **8 unit-тестів**:

1. Перевірка збереження замовлення
2. Перевірка відправки email
3. Перевірка помилки при null
4. Перевірка помилки при неправильній сумі
5. Перевірка отримання замовлення
6. Перевірка виклику репозиторію
7. Перевірка одного виклику Save
8. Перевірка одного виклику EmailService
